# Kuji Info

애니메이션, 게임, 캐릭터 IP 기반의 굿즈/이벤트/쿠지 정보를 아카이빙하고 관리하기 위한 프로젝트입니다.

공식 가격, 판매 이벤트, 쿠지 라인업, 굿즈 구성품을 구조화하여 저장하고 조회할 수 있도록 개발했습니다.

---

## 프로젝트 소개

중고 거래 플랫폼에서 굿즈를 구매할 때 공식 가격이나 판매 출처를 확인하기 어려운 경우가 많습니다.

특히 다음 정보를 한곳에서 확인하기 어렵습니다.

- 원래 정가가 얼마였는지
- 어떤 이벤트에서 판매되었는지
- 어떤 쿠지의 몇 상 경품인지
- 랜덤/선택형 굿즈의 구성품이 무엇인지
- 비매품 또는 특전인지

이 프로젝트는 IP를 기준으로 이벤트, 쿠지, 굿즈 정보를 정리하고, 굿즈의 출처를 이벤트 또는 쿠지 라인업과 연결해 관리하기 위해 개발했습니다.

---

## 주요 기능

### IP 관리

- IP 등록 / 수정 / 삭제
- 공식 링크 및 설명 관리
- IP 기준 이벤트, 쿠지, 굿즈 분류

### 이벤트 관리

- IP 이벤트 등록 / 수정 / 삭제
- 이벤트 기간, 장소, 공식 링크, 설명 관리
- 이벤트와 굿즈 연결

### 쿠지 관리

- 쿠지 등록 / 수정 / 삭제
- 일본/국내 회당 가격, 출시일, 총 수량 관리
- 쿠지 라인업 등록 / 수정 / 삭제
- A상부터 L상, 라스트원상 등급 선택
- 굿즈가 생성되기 전에도 라인업을 먼저 등록 가능

### 굿즈 관리

- 굿즈 등록 / 수정 / 삭제
- 공식 가격 및 비매품 여부 관리
- 제조사, 굿즈 종류, 판매 방식 관리
- 썸네일 이미지, 공식 링크, 설명 관리
- 이벤트 또는 쿠지 라인업 중 하나와 연결
- 굿즈 목록에서 IP, 제조사, 이벤트, 쿠지, 판매 방식, 비매품 여부, 굿즈 종류 필터링

### 굿즈 구성품 관리

- 굿즈 아이템 등록
- 단일 / 선택 구매 / 랜덤 판매 방식 지원
- 캐릭터명, 이미지, 레어도, 확률 관리

## 서비스 흐름

### 이벤트 굿즈

```text
IP 등록
→ 이벤트 등록
→ 굿즈 등록
→ 굿즈를 이벤트에 연결
```

### 쿠지 경품

```text
IP 등록
→ 쿠지 등록
→ 쿠지 라인업 등록
→ 굿즈 등록
→ 굿즈를 기존 쿠지 라인업에 연결
```

이벤트와 쿠지는 굿즈 기준에서 동시에 선택할 수 없습니다. 하나의 굿즈는 이벤트 출처이거나 쿠지 라인업 출처 중 하나로 관리합니다.

## 데이터 구조

```text
Ip
 ├── IpEvent
 │    └── EventGoods
 │         └── Goods
 │              └── GoodsItem
 │
 └── Kuji
      └── KujiLineup
           └── Goods?
                └── GoodsItem
```

`KujiLineup.goodsId`는 선택값입니다. 그래서 쿠지 라인업은 굿즈 상세가 생성되기 전에도 먼저 등록할 수 있고, 나중에 굿즈 등록/수정 화면에서 연결할 수 있습니다.
